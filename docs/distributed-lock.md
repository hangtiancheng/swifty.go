# 分布式锁

- 独占性: 一个锁最多只能被一个竞争者占有
- 健壮性: 不能死锁, 占有者崩溃后必须自动释放
- 对称性: 加锁和解锁必须是同一个身份
- 高可用: 分布式锁服务组件, 必须有故障转移能力

## 分布式锁的分类

- 主动轮询: CAS + 自旋, 锁竞争者持续请求获取锁, 即使锁已被占用, 直到获取锁成功
- 监听回调: 阻塞 + 唤醒, 锁竞争者发现锁已被占用时, 创建 listener 监听器, 定于锁释放事件; 之后不再持续请求获取锁; 直到锁释放事件发布, 再重新请求获取锁

自旋锁: 使用 CAS (Compare And Swap)

| 锁竞争                | 特点                           | 适用场景   |
| --------------------- | ------------------------------ | ---------- |
| 主动轮询: CAS + 自旋      | 不阻塞协程, 没有上下文切换开销 | 并发不激烈 |
| 监听回调: 阻塞 + 唤醒 | 阻塞协程, 有上下文切换开销     | 并发激烈   |
